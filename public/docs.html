1. Resumen

Esta aplicación es un demo académico que permite ingresar un nombre de usuario de Instagram, abrir su página pública con un navegador controlado por Playwright (Chromium en modo headless) y devolver metadatos visibles del perfil junto con hasta N enlaces a publicaciones detectadas en la rejilla (por defecto se pide un mínimo de 10).

Estrategia en dos capas. Primero se intenta en modo anónimo (sin cookies de sesión). Si Instagram muestra un muro de inicio de sesión y no hay publicaciones en la rejilla, el sistema puede reintentar usando un archivo de sesión (storage state) generado con npm run auth:setup.

2. Objetivos y alcance

  • Demostrar web scraping sobre una SPA moderna sin Selenium ni BeautifulSoup.
  • Implementar un flujo usuario → validación → extracción → presentación en una interfaz web clara.
  • Incorporar el uso opcional de cookies vía Playwright storageState.
  • Documentar limitaciones técnicas y consideraciones de privacidad / ToS.

No se incluye descarga masiva de medios (imágenes/videos), ni bypass de captchas, ni rotación de proxies: el foco es un trabajo de ingeniería reproducible y defendible.

4. Flujo anónimo → cookies

1
Contexto limpio

Se abre Chromium sin storageState: equivalente a un visitante sin sesión guardada.

2
Navegación al perfil

URL directa https://www.instagram.com/<usuario>/ (no hace falta “buscar” en el buscador de Instagram).

3
Scroll y extracción

Se hace scroll moderado para cargar más celdas de la rejilla y se recolectan enlaces /p/<shortcode>/.

4
Reintento con sesión (opcional)

Si el resultado es LOGIN_WALL y existe el archivo de sesión configurado, se repite el mismo proceso inyectando cookies mediante storageState.

5. Instalación y ejecución

Requisitos

  • Node.js 18 o superior.
  • Conexión a Internet la primera vez (descarga de Chromium por Playwright).

Comandos

cd webscraping
npm install
npm start

Luego abrí http://localhost:3000 (o el puerto definido en .env).

Instalación manual del navegador (si falla postinstall)

npx playwright install chromium

6. Cookies y sesión segura

Las cookies de sesión son credenciales. Este proyecto las guarda solo en un archivo local ignorado por Git, por ejemplo instagram-auth.json.

Generar el archivo de sesión

npm run auth:setup
  1. Se abre Chromium en modo visible.
  2. Iniciás sesión en Instagram manualmente.
  3. Volvés a la terminal y presionás Enter.
  4. Se escribe instagram-auth.json (o la ruta de STORAGE_STATE_PATH).
Nunca subas instagram-auth.json ni captures de pantalla con valores de cookies al repositorio o al informe. Si se filtran, cerrá sesión en Instagram y regenerá el archivo.

Variable de entorno

Copiá .env.example a .env y ajustá si usás otra ruta:

STORAGE_STATE_PATH=./instagram-auth.json
PORT=3000

7. API REST

POST /api/scrape

Cuerpo JSON:

{
  "username": "usuario",
  "minPosts": 10
}

Respuesta exitosa (ejemplo parcial):

{
  "ok": true,
  "username": "usuario",
  "profileUrl": "https://www.instagram.com/usuario/",
  "mode": "anonymous",
  "profile": { ... },
  "posts": [ "https://www.instagram.com/p/XXXX/" ],
  "stats": { "requestedMinPosts": 10, "collectedPosts": 12 }
}

Otros endpoints

  • GET /api/health — comprobación del servicio.
  • GET /api/validate-username?u=nombre — validación básica del formato.

8. Datos extraídos

Campo Descripción
profile.fullName Nombre mostrado si aparece en el encabezado.
profile.biography / meta Texto de bio o descripción meta cuando es visible.
profile.*Label Cadenas de seguidores / seguidos / publicaciones si el DOM las expone.
posts[] URLs de publicaciones encontradas en la rejilla (hasta el máximo configurado).

Instagram cambia el HTML con frecuencia: algunos campos pueden venir vacíos aun cuando el perfil es público. Eso es normal en scraping y debe explicarse en el informe.

9. Errores y advertencias

Código Significado
NOT_FOUND Usuario inexistente o página no disponible.
PRIVATE Cuenta privada: no hay rejilla pública.
LOGIN_WALL Instagram exige login o bloquea la vista; usar sesión con cookies.
FEW_POSTS Advertencia: menos publicaciones de las solicitadas.

10. Ética, legalidad y buenas prácticas

  • Usá solo perfiles públicos y datos que un visitante podría ver en la web.
  • Respetá los Términos de uso de Instagram y las políticas de tu universidad.
  • Mantené baja frecuencia de solicitudes; este proyecto no está pensado para uso masivo.
  • En la defensa oral, contrastá scraping vs API oficial (Graph API) como alternativa empresarial.
